# 피드백
1. js diff version up, time 명령어로 시간 비교
2. PR 할 때 마스터로 하지말자. 
 - git checkout -b fixthings
 - git push origin checkout
 이 후 이것으로 PR, PR이 merge가 되거나 또 다른 일들이 발생한 후의 master 브랜치는 업데이트 내용으로 쌓이게 되고
 그 쌓은 것을 다시 origin에 merge를 할 수 있다. 

 - git fetch
 - git merge origin/master 
 
 또는.. 

 - git pull

 로 할 수 있다. 
 위와 같은 식으로 git fetch + merge 또는 pull을 하면 된다. 
 (아싸님은 git fetch + merge를 하는 것이 좋다고 한다.)

 정리하자면 이런식으로 merge를 할 때는 최신 master 브랜치 위에 올려야 하고 올릴 때는 다른 브랜치(master가 아닌)를 통해 올려야 한다. 자신의 로컬에 있는 master는 최후의 보루, 그 다른 브랜치가 closed될 수 있으니 하나의 보루로 남겨 놓고 나중에 다른 PR을 올릴 때 자신의 로컬에 있는 master을 git fetch + merge를 통해 update를 해주면 된다.
